# PortManager GUI - 使用说明

## 核心特性

**现代化卡片式界面** - 参考 G-Helper 风格设计  
**开机自启动** - 下次重启自动运行  
**桌面窗口** - 原生软件界面，不是网页  
**轻量启动** - 直接打开应用窗口  
**端口管理** - 扫描、查看、关闭端口  

---

## 快速上手

### 步骤 1: 启动
```
双击 run.bat
或
以管理员身份运行 PortManager.exe
```

### 步骤 2: 等待窗口出现
程序会直接打开 PortManager 主窗口

### 步骤 3: 开始使用
- 点击"快速扫描"检测常用/高风险端口
- 点击"全面扫描"检测全部监听端口
- 查看每个端口的进程信息
- 点击"关闭"按钮关闭不需要的端口

## 常见操作

### 操作 1: 查看当前开放的所有端口

```
1. 启动 PortManager
2. 点击左卡片的"快速扫描"或"全面扫描"按钮
3. 等待 3-5 秒
4. 在中卡片查看结果
```

**显示内容**:
```
发现了 N 个开放端口
─────────────────
端口 8080
nginx.exe (PID: 1234)
[关闭 按钮]

端口 3306
mysqld.exe (PID: 5678)
[关闭 按钮]

... 更多端口 ...
```

### 操作 2: 关闭特定端口

```
1. 执行"操作 1"进行扫描
2. 在中卡片找到你想关闭的端口
3. 点击该条目右侧的"关闭"按钮
4. 确认关闭操作
5. 完成！端口已关闭
```

**警告**: 关闭端口会立即终止相关进程！

### 操作 3: 启用开机自启动

```
1. 打开 PortManager
2. 在右卡片找到"开机自启动"复选框
3. 勾选复选框（打钩）
4. 关闭程序
5. 下次重启时 PortManager 会自动启动
```

**提示**: 可随时取消勾选来禁用开机启动

### 操作 4: 查看版本信息

```
1. 右卡片点击"关于"按钮
2. 弹出版本号和功能说明
3. 点击"关闭"关闭对话框
```

## 重要提示

### 权限要求

某些端口操作需要 **管理员权限**:
- `run.bat` 会自动请求管理员权限
- 如果直接运行 exe，请以管理员身份运行
- 如果看到权限错误，尝试重新以管理员身份运行

### 系统关键端口

**不要关闭这些端口**:
```
系统端口（通常被 svchost.exe 占用）:
  - 135
  - 445  
  - 139
  - ...
  
关键应用服务:
  - Windows Update
  - RDP (远程桌面默认是 3389)
  - 其他系统服务
```

**后果**: 关闭系统端口可能导致:
- 网络连接中断
- 系统功能异常
- 远程连接失效

### 关闭端口的影响

**重要**:
- 关闭端口 = 终止占用该端口的进程
- 该应用会立即停止运行
- 如果应用依赖于该端口，可能无法重新启动
- **建议只关闭你能识别的应用端口**

## 故障排查

### 问题 1: 程序无法启动

**症状**: 双击 `run.bat` 或 `exe` 没有反应

**解决方案**:
1. 确保以管理员身份运行
2. 检查 Windows 防火墙是否阻止
3. 尝试从命令行运行查看错误信息
4. 检查是否有端口冲突（默认占用随机端口）

### 问题 2: 窗口没有出现

**症状**: 程序启动后没有看到主窗口

**解决方案**:
1. 确保程序没有在后台报错退出
2. 检查是否以管理员身份运行
3. 查看任务管理器中是否有 PortManager 进程

### 问题 3: 扫描端口很慢

**症状**: 扫描需要超过 10 秒

**解决方案**:
1. 这是正常现象（3-5 秒通常）
2. 如果特别慢，可能网络有问题
3. 尝试重新扫描
4. 检查系统是否有大量背景任务

### 问题 4: 关闭端口失败

**症状**: 点击"关闭"后没有反应或显示错误

**解决方案**:
1. 确保以管理员身份运行
2. 该端口可能已经关闭了，再次扫描确认
3. 该端口可能是系统关键端口（如 445）
4. 尝试在任务管理器中手动关闭对应进程

### 问题 5: 开机自启不工作

**症状**: 设置了开机自启但重启后没有启动

**解决方案**:
1. 确保勾选了"开机自启动"复选框
2. 重启电脑（不是注销）
3. 检查 Windows 启动项是否包含 PortManager
4. 某些杀毒软件可能阻止自启

## 使用建议

### 日常使用建议

**定期扫描**
- 每周扫描 1-2 次
- 检查是否有陌生的端口或进程
- 及时关闭不必要的端口

**启用开机自启**
- 勾选"开机自启动"
- 在后台保护你的系统
- 可随时禁用

**记住关键端口**
- 80: 常见 HTTP 服务
- 443: HTTPS
- 3306: MySQL
- 5432: PostgreSQL
- 6379: Redis

**谨慎关闭**
- 只关闭你认识的应用端口
- 关闭前了解该应用的用途
- 如有疑问，不要关闭

### 高级用法

**分析流量来源**：
通过 PID 从任务管理器找到占用端口的进程

**安全检查**：
定期扫描检查是否有恶意程序占用端口

**端口用途分析**：
根据进程名判断该端口的用途
