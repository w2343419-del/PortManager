# 快速开始指南

## ⚡ 30 秒快速开始

### 第一步：启动程序
```
双击 run.bat
```
或以管理员身份运行 `PortManager.exe`

### 第二步：等待窗口出现
- 程序会直接打开主窗口
- 显示现代化的卡片式界面

### 第三步：开始使用
1. **点击"扫描端口"** - 发现所有开放端口
2. **查看结果** - 了解每个端口的进程信息
3. **关闭端口** - 点击"关闭"按钮关闭不需要的端口

---

## 🔍 三个主要功能

### 功能 1：端口扫描
```
1. 点击左侧的"扫描端口"按钮
2. 等待扫描完成（通常 3-5 秒）
3. 查看扫描状态和发现的端口数量
```

### 功能 2：查看结果
```
中间卡片自动显示：
├─ 发现的端口总数（大字体显示）
├─ 每个端口的详细信息
│  ├─ 端口号
│  ├─ 关联进程名
│  └─ 进程 ID (PID)
└─ 每个端口的"关闭"按钮
```

### 功能 3：设置
```
右侧卡片提供：
├─ 开机自启动（勾选启用/禁用）
└─ 关于（查看版本信息）
```

## 💼 工作流示例

### 场景 A：清理不用的端口
```
1. 点击"扫描端口"
   ↓
2. 查看发现的所有端口
   ↓
3. 找到你想关闭的端口（比如 8080）
   ↓
4. 点击该端口行的"关闭"按钮
   ↓
5. 确认操作
   ↓
6. ✅ 完成！端口已关闭
```

### 场景 B：启用开机自启
```
1. 打开 PortManager
   ↓
2. 找到右卡片的设置区域
   ↓
3. 勾选"开机自启动"复选框
   ↓
4. ✅ 完成！下次重启会自动启动
```

### 场景 C：定期检查系统
```
每周定期：
1. 打开 PortManager
2. 扫描端口
3. 确认所有端口都是预期的
4. 关闭任何异常或未知的端口
```

## ⚠️ 重要提示

1. **需要管理员权限**
   - 关闭端口需要管理员权限
   - 使用 `run.bat` 会自动请求权限

2. **不要随意关闭系统端口**
   - 系统进程端口关闭会导致系统异常
   - 建议只关闭你能识别的应用端口

3. **关闭端口会终止进程**
   - 关闭端口意味着关闭占用该端口的程序
   - 某些应用可能需要这个端口才能运行

## 🆘 故障排查

### 问题：程序无法启动
**解决**:
- 检查是否需要管理员权限
- 尝试使用 `run.bat` 脚本
- 确保 Windows 防火墙未阻止

### 问题：扫描速度慢
**解决**:
- 这是正常现象（3-5 秒）
- 网络状况可能影响速度
- 尝试再次扫描

### 问题：关闭端口失败
**解决**:
- 确保以管理员身份运行
- 该端口可能是系统关键端口
- 尝试手动在任务管理器中关闭进程
